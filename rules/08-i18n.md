# Rule 08: i18n (Internationalization)

## 목적
페이지·컴포넌트·스키마 메시지에 흩어진 하드코딩 문자열을 키 기반으로 추출하여 다국어 카탈로그 한 곳에서 관리.

## 규칙 체크리스트

### 추출
- [ ] 사용자에게 보이는 모든 문자열은 `t('namespace.key')` 경유
- [ ] 키는 도메인 네임스페이스 (`login.*`, `post.*`, `comment.*`, `common.*`)
- [ ] zod 스키마의 사용자 향 메시지도 `t()` 사용
- [ ] placeholder, aria-label, button text 빠짐없이

### 카탈로그
- [ ] `messages/ko.ts`, `messages/en.ts` 같은 키 셋 유지
- [ ] 누락 키는 기본 locale로 fallback
- [ ] 파라미터 보간(`{name}`) 지원

### 회귀 방지
- [ ] 키 누락 시 컴파일 에러 (TypeScript `MessageKey` literal union)
- [ ] 기존 테스트 모두 통과 (기본 locale ko 유지)
- [ ] locale 스위치 단위 테스트 1개 이상

## 적용 대상
- `lib/i18n/messages.ts`, `lib/i18n/index.ts` (신규)
- `lib/schemas.ts`
- `app/login/page.tsx`, `app/posts/page.tsx`, `app/posts/new/page.tsx`, `app/posts/[id]/page.tsx`
- `components/CommentList.tsx`
