# Rule 07: Accessibility (a11y)

## 목적
폼·리스트·동적 영역에 ARIA 속성과 키보드 친화적 마크업을 일관 적용. 컴포넌트 만들 때 함께 적용되도록.

## 규칙 체크리스트

### 문서 구조
- [ ] `<html lang="ko">` (실제 언어와 일치)
- [ ] 페이지마다 `<h1>` 하나
- [ ] 의미 있는 영역은 `<section aria-label="...">` / landmark

### 폼
- [ ] 모든 입력은 `<label>`로 연결 (visible 또는 sr-only)
- [ ] 입력 ↔ 에러 메시지를 `aria-describedby`로 연결
- [ ] `aria-invalid={errors.X ? 'true' : 'false'}`
- [ ] 에러 영역은 `role="alert"` 또는 `aria-live="assertive"`
- [ ] 제출 중에는 `aria-busy="true"`

### 동적 영역
- [ ] 로딩 상태는 `role="status"` + `aria-live="polite"`
- [ ] 비파괴 알림은 `aria-live="polite"`

### 인터랙션
- [ ] 아이콘만 있는 버튼은 `aria-label` 필수
- [ ] 같은 텍스트 버튼이 여러 개면 어떤 항목 대상인지 `aria-label`에 포함
- [ ] 키보드만으로 모든 동작 가능 (네이티브 button/input/link 우선)

### 시각
- [ ] 색상에만 의존하지 않음 (에러는 텍스트로도 전달)
- [ ] 포커스 표시 유지 (글로벌 `outline: none` 금지)

## 검증 방법

```bash
# label 누락 입력 찾기 (대략)
grep -rn "<input\|<textarea" app/ components/ --include="*.tsx" | grep -v "aria-label\|id="

# 한국어 lang 설정
grep -n 'lang=' app/layout.tsx
```

## 적용 대상
- `app/layout.tsx`
- `app/login/page.tsx`
- `app/posts/new/page.tsx`
- `app/posts/page.tsx` (개별 삭제 버튼)
- `components/CommentList.tsx`
