# AI Quality Baseline

> "중요한 건 알지만 시간이 없어서 타협했던 것들"  
> AI가 속도 비용을 없애줬으니, 이제 이것들을 안 챙기는 건 변명이 아니라 **의지 문제**다.

---

## 개념

기존에는 품질을 위한 작업들이 **속도 트레이드오프**를 요구했다.  
AI는 그 속도 비용을 0에 가깝게 만들었다.  
따라서 아래 항목들은 이제 **기본값(default)**이 된다.

---

## 목록

### 코드 품질

| 항목 | 원래 단점 | AI 상쇄 방법 |
|------|----------|-------------|
| **테스트 코드** | 작성 시간이 오래 걸림 | 스펙 주면 테스트 뼈대 자동 생성 |
| **TypeScript 타입 정의** | interface/type 작성 귀찮아서 `any` 남발 | API 응답 보여주면 타입 즉시 생성 |
| **에러 핸들링** | 모든 엣지케이스 처리 번거로워 happy path만 | "에러 케이스 다 처리해줘" 한 마디로 커버 |
| **리팩터링** | 동작하는 코드 건드리기 무서움 (터지면 책임) | 리팩터링 + 테스트 동시 요청 → 안전망 확보 |
| **보일러플레이트 제거** | 새 페이지/컴포넌트마다 반복 구조 복붙 | 패턴 한 번 보여주면 일관된 구조로 계속 생성 |

### 문서화

| 항목 | 원래 단점 | AI 상쇄 방법 |
|------|----------|-------------|
| **코드 문서화** | 코드 짜고 나서 주석/README 쓸 여력 없음 | 코드 주면 문서 자동 생성 |
| **Storybook / 컴포넌트 문서화** | 컴포넌트 만들고 Story 따로 쓰는 게 이중 작업 | 컴포넌트 코드 주면 Story 자동 생성 |

### 보안 & 품질 검증

| 항목 | 원래 단점 | AI 상쇄 방법 |
|------|----------|-------------|
| **코드 리뷰** | 사람 리뷰어 시간에 의존 | PR 올리기 전 AI 사전 리뷰 |
| **보안** | XSS, CSRF, 인증 토큰 처리 등 챙길 게 많아서 대충 넘김 | 코드 보여주면 취약점 스캔 + 수정안 즉시 |

### 사용자 경험

| 항목 | 원래 단점 | AI 상쇄 방법 |
|------|----------|-------------|
| **접근성 (a11y)** | aria 속성, 키보드 네비게이션 챙기기 번거로움 | 컴포넌트 만들 때 같이 적용 요청 |
| **국제화 (i18n)** | 하드코딩된 텍스트 다 뽑아서 key로 바꾸는 작업이 노가다 | 컴포넌트 주면 i18n key 추출 + 파일 구조까지 자동 |
| **로깅 / 모니터링 연동** | Sentry, GA 이벤트 심는 게 귀찮아서 주요 흐름만 | "모든 유저 액션에 로깅 추가해줘" 한 번에 |

## 적용 계층

항목 성격별로 **언제 챙길 것인지**를 나눈다.

| 시점 | 항목 |
|------|------|
| **항상 (코드 작성 시)** | 타입 정의, 에러 핸들링, 보안 |
| **컴포넌트 생성 시** | a11y, Storybook Story, i18n key |
| **PR 올리기 전** | 코드 리뷰, 테스트, 로깅 |
| **프로젝트 초기 설정 시** | 보일러플레이트 패턴, 문서화 구조 |

---

## 핵심 원칙

1. **기본값이 바뀌었다** — 이 목록은 "하면 좋은 것"이 아니라 "안 하면 설명해야 하는 것"이다.
2. **속도 핑계는 없다** — AI가 속도 비용을 없앴다. 이제 트레이드오프 계산이 달라졌다.
3. **의지가 아니라 시스템으로** — CLAUDE.md 규칙, hooks, 체크리스트로 자동화해서 의지에 의존하지 않는다.
